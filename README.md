## Про проект

Реалізовано чотири версії алгоритму множення матриць **C = A × B**:

| Версія | Файл | Парадигма | Масштабованість |
|--------|------|-----------|-----------------|
| Послідовна | `matmul_seq.c` | — | 1 ядро |
| OpenMP | `matmul_omp.c` | Shared Memory | до ~64 ядер |
| MPI | `matmul_mpi.c` | Distributed Memory | до тисяч вузлів |
| Гібридна | `matmul_hybrid.c` | MPI + OpenMP | кластери з багатоядерними вузлами |

Кожна реалізація містить **вбудоване профілювання накладних витрат** з покомпонентним виводом.

## Алгоритми та оптимізації

### Оптимізація кешування — транспонування B

У класичному алгоритмі доступ до стовпців B спричиняє cache misses (row-major пам'ять):

```
Класичний:        C[i][j] += A[i][k] * B[k][j]    ← B по стовпцях cache miss
З транспонуванням: C[i][j] += A[i][k] * BT[j][k]   ← обидва по рядках cache hit
```

Прискорення лише від цієї оптимізації: **~1.8× для n=1024**.

### Рядкова декомпозиція (MPI)

```
Матриця A          MPI_Scatter        Rank 0  → обчислює рядки C[0..rows-1]
──────────         ──────────►        Rank 1  → обчислює рядки C[rows..2rows-1]
Rank 0 рядки                          Rank 2  → обчислює рядки C[2rows..3rows-1]
Rank 1 рядки       MPI_Bcast(B)       Rank 3  → обчислює рядки C[3rows..n-1]
Rank 2 рядки       ──────────►                          │
Rank 3 рядки       B на всіх ranks             MPI_Gather(C)
                                                        ↓
                                               Повна матриця C
```

### Fork-join модель (OpenMP)

```
Головний потік
      │
      ▼  #pragma omp parallel for schedule(static)
      ├── Thread 0 → рядки  0 .. n/t-1
      ├── Thread 1 → рядки  n/t .. 2n/t-1
      ├── Thread 2 → рядки  2n/t .. 3n/t-1
      └── Thread t-1 → рядки  (t-1)n/t .. n-1
                │
          неявний бар'єр
                │
           Матриця C
```

### NUMA-коректна ініціалізація (first-touch)

```c
/* Погано: всі сторінки виділяються на сокеті 0 */
for (int i = 0; i < n*n; i++) A[i] = ...;

/* ✓ Добре: кожен потік виділяє свої сторінки поруч зі своїм ядром */
#pragma omp parallel for schedule(static)
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        A[i*n+j] = ...;
```

### Усунення false sharing

```c
/* n записів у спільну кеш-лінію → інвалідація у всіх потоків */
for (int k = 0; k < n; k++)
    C[i*n+j] += A[i*n+k] * BT[j*n+k];

/* ✓ 1 запис в кінці → false sharing виключено */
double sum = 0.0;
for (int k = 0; k < n; k++)
    sum += A[i*n+k] * BT[j*n+k];
C[i*n+j] = sum;
```

## Структура репозиторію

```
.
├── matmul_seq.c              # Послідовний: naive + cache-opt, порівняння
├── matmul_omp.c              # OpenMP: паралельний for, профілювання fork/join

├── Makefile                  # make all / test / benchmark

```

## Запуск

```bash
# Послідовний (порівнює naive vs кеш-оптимізований)
./matmul_seq 1024

# OpenMP — 8 потоків
./matmul_omp 1024 8

# MPI — 8 процесів
mpirun -np 8 ./matmul_mpi 1024

# Гібрид — 4 MPI × 3 OpenMP = 12 ядер (оптимальна конфігурація)
mpirun -np 4 ./matmul_hybrid 4096 3

# Тест (n=512, всі версії)
make test

# Бенчмарк (n=1024, всі конфігурації)
make benchmark
```

| 2     | 1.95  | 97.5%   | 1.98     | 99.0%    | —        |

| 8     | 6.74  | 84.3%   | 7.41     | 92.6%    | —        |
| 12    | 8.44  | 70.3%   | 10.53    | 87.7%    | **10.80**|

### S(p) при n=4096 (NUMA-ефект)

| p=t=12 | MPI    | OpenMP | Гібрид 4×3     |
|--------|--------|--------|----------------|
| S(p)   | 11.21  | 10.15  | **11.96**      |

> При великих матрицях MPI перевершує OpenMP через NUMA-ізоляцію пам'яті.

## Накладні витрати — масштабованість (p=8)

| n    | НВ (мс) | Обчислення (с) | Частка НВ |
|------|---------|----------------|-----------|
| 512  | 32      | 0.44           | 6.8%      |
| 1024 | 124     | 1.10           | **10.2%** |
| 2048 | 480     | 7.95           | 5.7%      |
| 4096 | 1860    | 62.0           | 2.9%      |

НВ зростають як **O(n²)**, обчислення — як **O(n³)** → відносна частка НВ спадає.

## Практичні рекомендації

| Умова | Рекомендація |
|-------|-------------|
| n < 1000, один вузол | **OpenMP** — мінімальні НВ (0.5%) |
| n > 3000, один вузол | **MPI** — NUMA-ізоляція перекриває комунікації |
| Кластер, будь-який n | **Гібрид MPI+OpenMP** |
| Обмежені ресурси розробки | **OpenMP** — простіше налагодження |
